# SDD 规范驱动开发工作流

## 一、什么是 SDD

SDD = Spec-Driven Development（规范驱动开发）。

核心理念是：先写规范，再写代码。规范是人和 AI 共同协作时的唯一事实来源。

## 二、核心工作流

```text
/specify -> /plan -> /tasks -> /implement
   什么        怎么        任务         实现
```

| 阶段 | 命令 | 产出物 | 说明 |
| :--: | :-- | :-- | :-- |
| 1 | `/specify` | `spec.md` | 明确用户故事、功能需求、验收标准 |
| 2 | `/plan` | `plan.md` | 明确架构设计、数据库方案、接口定义 |
| 3 | `/tasks` | `tasks.md` | 拆成可执行任务清单，通常带 checkbox |
| 4 | `/implement` | 源代码 | AI 根据前三阶段产物实现代码 |

## 三、完整开发流程

基于当前项目实践，可以整理为 6 步全栈流程：

| 阶段 | 任务 | 输入 | 输出 | 说明 |
| :--: | :-- | :-- | :-- | :-- |
| 1 | 设计数据库 | 需求、用户故事 | 数据库设计文档、DDL | 明确表结构、字段、索引、关系 |
| 2 | 生成 AI 指令文档 | 需求、模板 | `copilot-instructions.md` | 约束 AI 行为、代码规范、安全要求 |
| 3 | 设计接口文档 | 数据库、需求 | API 接口文档 | 推荐使用 OpenAPI / Swagger |
| 4 | 设计后端架构 | 接口文档 | 后端目录结构文档 | 明确 Controller / Service / DAO 分层 |
| 5 | 实现后端代码 | 接口、架构、指令 | 完整后端代码 | 按任务逐个实现 |
| 6 | 实现前端代码 | 接口、需求、指令 | Vue 前端代码 | 按用户故事逐个实现 |

如果希望强调输入输出关系，也可以按下面这个顺序理解：

1. 需求驱动数据库设计。
2. 需求同时驱动 AI 指令文档生成。
3. 数据库设计和需求一起驱动接口设计。
4. 接口设计继续驱动后端架构和前端实现。
5. 后端架构最终落到后端代码实现。

## 六、SDD 核心原则

### 1. 文档先行

- 不写一行代码，先写完规范文档。
- 文档就是设计，代码只是设计的实现。

### 2. 唯一事实来源

- 规范文档是唯一真相来源。
- 代码变更应先更新规范，再修改实现。

### 3. 逐层递进

- 数据库 -> 接口 -> 架构 -> 后端 -> 前端。
- 上一步输出，是下一步输入。

### 4. AI 可执行

- 规范足够详细时，AI 可以直接参与生成代码。
- 通过 `copilot-instructions.md` 统一约束 AI 输出。

### 5. 任务颗粒度可控

- 每个任务最好控制在 10 到 30 分钟内可完成。
- 任务可以按用户故事、模块或接口拆分。

## 七、copilot-instructions.md 模板

```md
# Copilot 指令文档

## 一、代码风格
- 使用 2 空格缩进
- 类名使用 PascalCase
- 方法名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE

## 二、后端规范
- 分层结构：Controller -> Service -> Mapper
- 使用 DTO 进行数据传输，不直接暴露 Entity
- 所有接口需要 Swagger 注解
- 统一返回格式：{ code, message, data }

## 三、前端规范
- 使用 Vue 3 + Composition API
- API 调用统一封装在 api/ 目录
- 组件使用 PascalCase 命名
- 状态管理使用 Pinia

## 四、安全要求
- 所有接口需要认证（除登录、注册外）
- 敏感数据不记录到日志
- 输入参数必须校验

## 五、AI 行为约束
- 每次只实现一个任务
- 生成代码后自动补充必要注释
- 不确定的地方先询问
```

## 八、SDD vs 其他方法

| 对比维度 | 传统开发 | Vibe Coding | SDD |
| :-- | :-- | :-- | :-- |
| 核心产物 | 需求文档 -> 代码 | 自然语言对话 | 规范（Spec） |
| 工作流程 | 需求 -> 设计 -> 编码 -> 测试 | 聊天式探索 | 规范 -> 计划 -> 任务 -> 实现 |
| 代码来源 | 人工编写 | AI 根据对话生成 | AI 根据规范生成 |
| 质量保障 | 人工评审、测试 | 质量波动较大 | 系统化验证 |
| 适用场景 | 大型稳定系统 | 原型、试验 | 生产系统、团队协作 |

## 九、常用命令速查

| :-- | :-- |
| `/specify` | 编写功能规范，如用户故事、验收标准 |

| `/fix` | 修复代码问题 |
| `/tests` | 生成单元测试 |

## 十一、快速开始

1. 编写需求：`docs/spec/user-stories.md`
2. 生成指令：`docs/copilot-instructions.md`
3. 设计数据库：`sql/schema.sql`
4. 设计接口：`docs/plan/api-design.yaml`
5. 设计架构：`docs/plan/backend-structure.md`
6. 拆解任务：`docs/tasks/tasks.md`
7. 开始实现：按任务清单逐个执行 `/implement`
