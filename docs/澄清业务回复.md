## 一、数据定义类
### Q01 🔴 序列号的格式与规则
**问题**
1. 序列号是纯数字，还是包含字母前缀？
2. 不同船司的序列号格式是否统一？
3. 序列号长度是否固定？
4. 起始号与结束号是否必须连续？
5. 字符串格式的排序规则是什么？

**业务方回答**
1. 序列号为**字母前缀+纯数字**组合，格式：**船司代码-8位连续数字**（例：COSCO-00012345）。
2. 不同船司格式统一，仅船司前缀不同。
3. 总长度固定：船司代码(3-4位)+横杠+8位数字。
4. 起始号与结束号必须连续，不允许间断。
5. 排序规则：先按船司前缀排序，再按数字段升序。

---

### Q02 🔴 是否按船司维度隔离管理
**问题**
1. 不同船司提单是否需要独立管理？
2. 入库是否必选船司？
3. 操作是否支持按船司筛选？

**业务方回答**
1. 必须按船司维度严格隔离，禁止跨船司冲突。
2. 入库时必选船司，系统自动拼接前缀。
3. 查询、编辑、作废、回收均支持按船司筛选。

---

## 二、状态机与流转类
### Q03 🔴 "已归档"状态如何触发
**问题**
1. 已归档的定义与触发方式？
2. 具体触发条件？
3. 归档后是否支持操作？
4. 归档粒度是号段还是单条？

**业务方回答**
1. 号码段内所有序列号全部使用完毕，系统自动归档。
2. 触发条件：已使用数量 + 作废数量 = 总数量。
3. 归档后支持查询，禁止回收、作废、编辑。
4. 以整个号码段为单位归档。

---

### Q04 🔴 回收后序列号状态变更
**问题**
1. 回收后状态变更为什么？
2. 是否立即可用？
3. 已归档是否可回收？
4. 批量回收上限？

**业务方回答**
1. 回收后状态变为已入库，直接释放复用。
2. 回收后立刻可用，无需确认流程。
3. 已归档序列号不允许回收。
4. 支持批量回收，单次上限50条。

---

### Q05 🔴 号码段交叉重叠处理规则
**问题**
1. 重叠/交叉是否允许保存？
2. 作废号段是否释放区间可重新录入？

**业务方回答**
1. 部分交叉/完全重叠均不允许保存；紧邻不重叠允许。
2. 已作废号码段自动释放区间，允许重新录入。

## 三、业务规则类
### Q06 🔴 作废原因枚举值
**问题**
1. 作废原因固定选项？
2. 是否支持自定义填写？
3. 是否纳入导出清单？
4. 是否需要二次确认？

**业务方回答**
1. 枚举：打印错误、纸张污损、丢失、船司回收、信息错误、其他。
2. 选择其他时支持自定义填写。
3. 作废原因纳入导出作废清单。
4. 作废必须二次确认弹窗。

---

### Q07 🔴 已使用号码段是否允许编辑
**问题**
1. 已使用是否可修改起止号？
2. 允许编辑哪些字段？
3. 是否记录操作日志？

**业务方回答**
1. 已使用不允许修改起始/结束号段。
2. 仅允许编辑备注、入库日期。
3. 编辑必须记录操作日志。

---

### Q08 🟡 查询条件必填逻辑
**问题**
1. 查询必填规则以哪个为准？
2. 无条件查询如何处理？
3. 分页默认值？

**业务方回答**
1. 提单号/使用日期/入库日期三选一必填。
2. 无条件查询提示：请至少输入一个查询条件。
3. 列表默认分页20条/页。

---

### Q11 🟡 回收范围限制
**问题**
1. 哪些状态可回收？
2. 已放单是否可回收？
3. 是否需要回收原因？
4. 是否通知相关方？

**业务方回答**
1. 仅已使用可回收，已归档不可回收。
2. 已放单禁止回收。
3. 回收无需填写原因。
4. 不通知客户，内部同步放单岗。

### Q10 🟡 导出作废清单格式与内容
**问题**
1. 导出格式？
2. 包含字段？
3. 导出范围？
4. 是否需要签名栏？

**业务方回答**
1. 格式：Excel，无固定模板。
2. 字段：船司、序列号、状态、作废原因、作废日期、操作人、入库日期。
3. 导出当前筛选结果。
4. 底部增加操作人签字、船司签收、日期栏。

## 五、权限与安全类
### Q12 🟡 操作权限控制
**问题**
1. 各岗位权限？
2. 作废回收是否需要审批？
3. 是否全量日志？
4. 防重复提交？

**业务方回答**
1. 制单岗：全操作；主管/审核岗：只读。
2. 作废、回收无需主管审批。
3. 所有操作全量留痕日志。
4. 保存按钮点击后禁用3秒。
