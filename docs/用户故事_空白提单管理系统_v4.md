### US001: 空白提单号码段入库

**用户故事**

> 作为 用户，
> 我希望 按船司录入空白提单号码段（起始号+结束号），系统自动生成该号段内所有序列号并统一管理，
> 以便 建立空白提单的数字化台账，减少100%的手工Excel登记工作（从每批5分钟缩短至1分钟内完成），且入库响应时间≤2秒。

**功能要点**

- 核心路径（必做）：选择船司（下拉必选）→ 输入起始号（8位数字）→ 输入结束号（8位数字）→ 系统自动计算总数量 → 填写备注（选填）→ 选择入库日期（默认今日）→ 点击「保存」→ 系统校验号码段 → 生成序列号列表
- 异常路径（必做）：① 起始号格式非8位纯数字时，实时提示"请输入8位纯数字"；② 号码段与已有号段部分交叉时，提示"号码段与已有号段（xxx~xxx）存在交叉，请修改"；③ 结束号 ≤ 起始号时，提示"结束号必须大于起始号"
- 边界条件（可选）：单个号码段最大支持 99999 张（起始号~结束号跨度≤99999）

**业务规则**

- 强制规则（不可协商）：
  - 序列号格式：`船司代码-8位连续数字`（如 COSCO-00012345），船司代码 3~4 位
  - 入库时**必选船司**，系统自动拼接前缀，入库人无需手动输入
  - 起始号与结束号必须连续，不允许间断
  - 同一船司下，新录入号段与已有号段不允许部分交叉或完全重叠；紧邻不重叠允许
  - 已作废号码段自动释放区间，允许重新录入
  - 保存按钮点击后禁用 3 秒防止重复提交
- 柔性规则（可协商）：
  - 备注默认为空，支持自由输入
  - 入库日期默认填充当日日期，可手动修改

**UI/交互设计约束**

**页面上下文**

- 所属页面：空白提单管理-入库弹窗（从列表页触发）
- 页面层级：`文件中心 > 首页 > 个人任务 > 空白提单管理` → 入库弹窗（二级弹窗）
- 入口位置：空白提单管理列表页顶部操作区「新增」按钮（主按钮，背景 #1677FF，文字 #FFFFFF，高度 32px，圆角 4px）
- 退出/返回逻辑：保存成功后关闭入库弹窗，自动刷新列表页；点击「取消」关闭弹窗不保存

**核心交互行为**

- 操作控件：
  - 船司选择：下拉选择器（单选，高度 32px，宽度 200px），选项来源系统船司主数据，支持模糊搜索
  - 起始号/结束号：单行文本输入框（高度 32px，宽度 200px，仅允许输入数字，最大长度 8 位）
  - 数量展示：文本标签（字号 14px，颜色 #1677FF，加粗），格式"共 {n} 张"，紧跟结束号输入框右侧（margin-left: 12px），跟随结束号输入实时更新
  - 备注输入：多行文本输入框（高度 80px，最大长度 200 字）
  - 入库日期：日期选择器（下拉式，高度 32px，默认今日，不可选未来日期）
  - 保存按钮：主按钮（背景 #1677FF，文字 #FFFFFF，边框 #1677FF，宽度 100%，高度 32px，圆角 4px，Hover #0F66E0）
  - 取消按钮：辅助按钮（背景 #FFFFFF，文字 #8C8C8C，边框 #E5E6EB，宽度 100%，高度 32px，圆角 4px，Hover #F5F7FA）
- 交互流程：
  1. 弹窗打开后，船司选择器默认获取焦点（下拉展开提示"请选择船司"）；
  2. 选择船司后，焦点自动跳转至起始号输入框；
  3. 输入起始号时，仅允许数字输入（非数字字符自动过滤）；
  4. 输入完起始号后 Tab 至结束号，输入结束号后系统实时计算并显示数量；
  5. 点击「保存」→ 触发号码段交叉校验 → 校验通过则显示加载状态 → 保存成功关闭弹窗；
  6. 点击「取消」→ 无确认弹窗，直接关闭。
- 焦点与操作逻辑：
  - 起始号/结束号未填写时，点击「保存」→ 输入框边框标红（#F53F3F）+ 底部显示"请输入起始号/结束号"；
  - 号码段交叉校验失败时，结束号输入框边框标红 + 底部显示交叉提示（含冲突号段信息）；
  - 支持 Tab 键切换焦点（船司→起始号→结束号→入库日期→备注→保存→取消）。

**视觉/布局约束**

- 核心元素布局：
  - 弹窗整体：居中弹窗，宽度 560px，高度自适应（最大高度 600px，超出显示滚动条），距顶部 15vh；
  - 弹窗遮罩：全屏遮罩，背景 rgba(0,0,0,0.45)，点击遮罩不关闭弹窗；
  - 弹窗头部：标题"空白提单入库"（字号 14px，常规字重，颜色 #333333，背景 #F5F7FA），右侧关闭按钮（×图标，24px，颜色 #86909C）；
  - 表单区域：单列布局，每个表单项垂直间距 16px，控件水平间距 12px，左右内边距 16px；
  - 标签样式：表单项标签居左，宽度 100px，字号 14px，颜色 #333333，必填项标签后标" *"（颜色 #F53F3F）。
- 控件样式规范：
  - 输入框：高度 32px，默认边框 #E5E6EB，聚焦边框 #1677FF（伴随 1px 阴影 0 0 0 2px rgba(22,119,255,0.1)），错误边框 #F53F3F；
  - 下拉框：高度 32px，边框 #E5E6EB，聚焦边框 #1677FF；
  - 数量标签：紧跟结束号输入框右侧（margin-left: 12px），字号 14px，颜色 #1677FF，加粗；
  - 提示文字：错误提示字号 12px，颜色 #F53F3F，位于输入框底部，距输入框 4px。
- 页面背景色：#F9FAFC；内容区背景：#FFFFFF
- 响应式要求：仅 PC 端（屏幕宽度≥1200px），不兼容平板/移动端。

**交互反馈规则**

- 加载状态：
  - 保存按钮点击后→ 按钮文本变为「保存中...」→ 右侧显示圆形加载动画（直径 14px，白色，旋转周期 1s）→ 按钮禁用（透明度 0.6）→ 持续禁用 3 秒防重复；
  - 号码段校验中→ 保存按钮显示加载状态。
- 成功反馈：
  - 保存成功→ 关闭弹窗→ 页面顶部 Toast 轻提示（3 秒后自动消失）；
  - 提示内容：成功图标 → 文本"入库成功！已生成 {n} 条序列号"（字号 14px）。
- 失败反馈：
  - 保存失败→ 弹窗顶部显示红色提示条（高度 32px，背景色 #FFF1F0）；
  - 提示内容：错误图标 → 文本"保存失败：{具体原因}"（字号 14px，颜色 #F53F3F）→ 右侧关闭按钮（×，16px）。

**设计资产关联**

- 组件库版本：企业组件库 v3.2.1（表单组件、弹窗组件、下拉选择器）

**界面要求**

- 船司选择器选中后，显示船司代码 + 船司名称（如"COSCO - 中远海运"）
- 起始号/结束号输入框右侧显示占位提示"例：00010001"（字号 12px，颜色 #8C8C8C）

**粒度评估**: M - 包含表单校验、号码段交叉校验、防重复提交、交互反馈，3-5 天可完成

**业务流程关联**

- 上游故事：无
- 下游故事：US002 空白提单号码段编辑
- 所属业务流程：入库→使用→回收/作废→归档

**最小可交付成果**：用户选择船司并录入号码段后，系统在 2 秒内完成校验并生成序列号记录，列表页实时刷新显示新入库记录。

**开发提示（业务层面）**

1. 号码段交叉校验逻辑：同船司下，检查 [start_no, end_no] 与所有已有未归档号段的 [exist_start, exist_end] 是否存在交集（NOT (end_no < exist_start OR start_no > exist_end)）
2. 已作废号段的区间释放：校验时排除状态为"已作废"且无序列号被使用的号码段
3. 起始号/结束号存储时保留原始字符串（带前导零），不做数值转换
4. 数量计算：total_qty = end_no - start_no + 1（数值比较）

**非功能需求**

- 性能：入库保存响应时间≤2秒；号码段校验响应时间≤500ms
- 安全：入库操作记录全量日志（操作人、时间、号码段、数量）；保存按钮禁用 3 秒防重复提交
- 兼容性：支持 Chrome 90+、Edge 90+、Firefox 88+

> 我希望 修改已入库号码段的备注和入库日期，

**功能要点**

- 强制规则：已使用号码段不允许修改起始/结束号段；仅允许编辑备注和入库日期；编辑操作必须记录操作日志（操作人、时间、修改前后值）

### US010: 操作日志审计查询

**用户故事**

> 作为 用户，
> 我希望 查看所有空白提单操作的完整日志记录（入库、编辑、使用、作废、回收、归档），
> 以便 满足合规审计需求，追溯操作责任，实现100%的操作可审计。

**功能要点**

- 核心路径（必做）：打开操作日志页面 → 按时间范围/操作类型/操作人筛选 → 查看日志列表 → 展开某条日志查看修改前后值
- 异常路径（必做）：筛选结果为空时展示空状态
- 边界条件（可选）：日志保留周期默认 1 年，超出自动归档

**业务规则**

- 强制规则：所有操作全量留痕（入库、编辑、使用、作废、回收、归档）；日志内容包含操作人、操作时间、操作类型、修改前值、修改后值
- 柔性规则：默认展示最近 7 天日志

**UI/交互设计约束**

**页面上下文**

- 所属页面：空白提单管理-操作日志页
- 页面层级：`文件中心 > 首页 > 个人任务 > 空白提单管理`，Tab 切换至「操作日志」
- 入口位置：空白提单管理页顶部 Tab「操作日志」
- 退出/返回逻辑：切换 Tab 回到其他视图

**核心交互行为**

- 操作控件：
  - 筛选区：时间范围选择器（高度 32px，默认近 7 天）、操作类型下拉（多选：入库/编辑/使用/作废/回收/归档，高度 32px）、操作人输入框（高度 32px，支持模糊搜索）、查询/重置按钮
  - 日志表格：列包含-操作时间、操作人、角色、操作类型、序列号/号码段、操作摘要、操作（展开详情）
    - 表头高度 40px，背景 #F5F7FA
    - 行高 40px，hover 高亮 #F0F7FF
    - 数字字段右对齐，文本字段左对齐
  - 展开行：点击「展开详情」→ 表格行下方展开灰色区域（背景 #F5F7FA），展示修改前值和修改后值的对比（差异部分标红）
- 交互流程：
  1. 页面加载默认查询近 7 天日志；
  2. 点击「展开详情」→ 行下方展开详情区域（动画 200ms）→ 展示修改前后值对比。

**视觉/布局约束**

- 页面背景 #F9FAFC，内容背景 #FFFFFF
- 筛选栏高度 40px，控件高度 32px，水平间距 12px，垂直间距 16px
- 表格：表头高度 40px，背景 #F5F7FA，字号 14px，颜色 #333333
- 展开详情区：padding 12px 16px，标题"变更详情"（字号 14px，颜色 #8C8C8C），修改前值和修改后值并排展示（各占 50%宽度），差异文本标红（background #FFF1F0，color #F53F3F）
- 页面边距：内容区左右内边距 16px

**交互反馈规则**

- 加载状态：表格骨架屏
- 空状态：插图 + "暂无操作日志"（字号 14px，颜色 #8C8C8C）

**设计资产关联**

- 组件库版本：企业组件库 v3.2.1（表格组件、可展开行）

**界面要求**

- Demo场景下默认所有功能按钮可见
- 日志表格默认按操作时间倒序

**粒度评估**: S - 标准日志查询页 + 展开详情，1-3 天可完成

**业务流程关联**

- 上游故事：所有涉及操作的故事
- 下游故事：无
- 所属业务流程：独立审计视图

**最小可交付成果**：用户打开操作日志页面后，系统在 2 秒内展示最近 7 天的操作日志，支持按条件筛选和展开查看变更详情。

**开发提示（业务层面）**

1. 操作日志表建议独立存储，不影响业务表性能
2. 修改前后值使用 JSON 格式存储，前端做 diff 高亮
3. Demo场景下不做角色权限拦截，默认登录用户可访问日志查询能力

**非功能需求**

- 性能：日志查询响应≤2秒；支持百万级日志数据量
- 安全：日志数据不可篡改（只追加，不修改/删除）
- 兼容性：支持 Chrome 90+、Edge 90+、Firefox 88+
